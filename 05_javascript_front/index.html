JavaScript (front-end) – bibliotecas para mobile, funcionalidades

Existem inúmeras bibliotecas desenvolvidas com JavaScript, porém há algumas que se destacam pela sua popularidade e, principalmente, por serem úteis para desenvolvimento mobile e funcionalidades interativas. Atualmente as bibliotecas que mais se destacam no mercado são o React.js, que é uma biblioteca JavaScript de código aberto mantida pelo Facebook (Meta), e o Angular, que é um framework JavaScript desenvolvido pelo Google.

Há também outras ferramentas menos populares se comparadas ao React e ao Angular, mas que são atuais e merecem uma atenção, como o Vue.js, que é um framework progressivo para construir interfaces de usuários. Ele é focado na camada de visualização apenas, facilitando a integração com outras bibliotecas ou projetos existentes. O Vue.js oferece um sistema reativo que facilita a criação de interfaces simples ou complexas de forma intuitiva e declarativa. Já o Next.js é um framework de JavaScript criado sobre o Node.js e o React, que permite aos desenvolvedores construir aplicações web estáticas e renderizadas pelo servidor (SSR – server-side rendering). O Next.js é projetado para oferecer uma experiência de desenvolvimento simplificada com desempenho otimizado e excelentes recursos de SEO (search engine optimization).

No entanto, existe também uma biblioteca muito popular chamada jQuery, que é uma das bibliotecas mais antigas em termos de desenvolvimento, porém sua praticidade auxilia muito em projetos novos e legados. O jQuery simplifica a manipulação do DOM (modelo de objeto de documento) e o tratamento de eventos e realiza chamadas AJAX (JavaScript e XML assíncronos). O jQuery, apesar de ser uma ferramenta antiga, continua sendo atual, prática e útil, então será abordada em um material à parte.

Neste conhecimento, serão abordadas mais especificamente as ferramentas React e Angular. Então, no próximo tópico, você conhecerá as principais características e funcionalidades dessas ferramentas e do JavaScript.

Desenvolvendo com React

Para iniciar o desenvolvimento de qualquer interface com React, diferentemente dos projetos tradicionais front-end, não é possível fazer a biblioteca funcionar diretamente no navegador; é preciso antes criar um ambiente de execução JavaScript, basicamente um servidor específico para rodar JavaScript. Para isso, será utilizado o Node.js.

Node.js

O Node.js, comumente chamado de Node, é uma plataforma de software de código aberto para a execução de código JavaScript no lado do servidor. Foi criado por Ryan Dahl em 2009 e é construído em cima do V8, que é o motor de JavaScript desenvolvido pelo Google para o navegador Chrome. O Node.js é projetado para construir aplicações de rede escaláveis e de alto desempenho, particularmente servidores web. Antes do Node.js, JavaScript era uma linguagem de programação voltada para client-side, executada apenas nos navegadores dos usuários. O Node.js expandiu o alcance do JavaScript, permitindo seu uso no lado do servidor, o que significa que os desenvolvedores podem escrever tanto o código do lado do cliente quanto do servidor em uma única linguagem

NPM (node package manager): o Node.js vem com o NPM, um gerenciador de pacotes que facilita a instalação, o compartilhamento e a atualização de pacotes de código. O NPM é uma parte essencial do ecossistema Node.js, fornecendo acesso a mais de 800 mil bibliotecas ou “pacotes” que podem ser facilmente integrados em projetos Node.js.

Apesar de ser popular para desenvolvimento web, o Node.js também é usado em uma variedade de outros contextos, incluindo automação de desktop, desenvolvimento de microsserviços, robótica e muito mais.

O Node.js transformou significativamente o desenvolvimento de software, permitindo uma abordagem mais unificada ao usar JavaScript em toda a stack (full stack) e continua sendo uma tecnologia fundamental para muitos desenvolvedores e empresas em todo o mundo.

Então, para começar a utilizar o React, é preciso instalar o Node em seu computador. Mas isso é uma tarefa bem simples: basta acessar o site oficial do Node.js, fazer seu download e instalar em seu computador como qualquer software comum.

Então, serão criados três pastas e alguns arquivos direto no diretório do seu projeto. Na pasta “node_modules” é onde ficam os arquivos com todas as dependências e configurações do seu ambiente de programação, basicamente um servidor JavaScript, com as configurações necessárias para o React funcionar. Os arquivos dessa pasta não devem ser modificados e são descartáveis, ou seja, eles não fazem parte do seu projeto em si. Sempre que você for colocar o seu projeto em React para rodar em outro computador ou servidor, será necessário reinstalar o ambiente para o React rodar.

Na pasta “public” é onde ficam arquivos que podem ou devem ser acessados diretamente, como imagens, arquivos de configuração do servidor, arquivos para comunicação com ferramentas de busca (como o robots.txt) etc.

Já a pasta “src” é a que mais será atualizada em um projeto React, pois nela ficam os arquivos com os dados em si, como conteúdos HTML (hypertext markup language), arquivos de CSS (cascading style sheets), imagens, entre outros.

Observe os arquivos que foram criados na raiz da pasta do seu projeto. Entre eles, haverá um arquivo chamado index.html, que não precisa ser modificado. No entanto, ao abri-lo, você perceberá que é esse arquivo HTML que, no final, renderiza o conteúdo. Todo o código que você adicionar dinamicamente com o React será mostrado dentro desse HTML.

Dentro da pasta “src”, você verá um arquivo chamado App.jsx, no qual fica o conteúdo principal do seu projeto. Nele, você terá um exemplo da sintaxe do JSX (JavaScript XML). Então, ao usar o React, você não cria mais conteúdos com o HTML, e sim com JSX, que basicamente é como se fosse o HTML, mas dentro do código de JavaScript. E o JSX traz algumas modificações e vantagens, pois, com ele, é possível interpolar variáveis e até mesmo criar funções, além de inserir valores em atributos de tags.

Se você domina bem HTML e JavaScript, achará a sintaxe do JSX bem intuitiva. Porém, há alguns detalhes importantes. No JSX, não é possível inserir uma classe em uma tag usando o atributo class. Para isso, no JSX, usa-se um atributo chamado className. Além disso, a cada função de return não se pode ter tags irmãs diretas, é sempre necessário ter uma tag superior em todo o conteúdo.

Então, se você precisar modificar o conteúdo da página principal do React, basta atualizar o arquivo App.jsx. No exemplo a seguir, apagou-se todo o conteúdo dentro da função App e adicionou-se o seguinte código: 

                      
function App() {   
 return ( 
  <div> 
  <h1 className="titulo">Olá, Mundo!</h1> 
  <p>Esse é o meu primeiro projeto com React!</p> 
  </div> 
  ) 
} 
export default App

Agora, você aprenderá a criar um componente no React. Qualquer interface para a web é construída com base em pequenas unidades, como botões, textos e imagens. O React permite combiná-las em componentes encaixáveis e reutilizáveis.

É possível pensar nos componentes como seções de um site que você poderá usar em diferentes partes da sua interface. Porém, cada componente não se limita apenas ao conteúdo HTML, podendo conter sua própria lógica e situações condicionais, aproveitando toda a flexibilidade que o JSX proporciona.

Para criar um componente, é preciso criar uma pasta chamada “components” dentro da pasta “src” do seu projeto. E dentro de “components”, cia-se um arquivo JSX para cada componente.

Por exemplo, veja a criação de um arquivo chamado Cursos.jsx que será um componente para mostrar uma lista de cursos – uma observação importante é que, no React, os nomes dos arquivos dos componentes precisam iniciar com letra maiúscula, por isso o nome Cursos.jsx, com “C” maiúsculo. Dentro desse arquivo, cria-se uma função muito similar ao exemplo visto no arquivo App.jsx: 

                      
function Cursos() {      
    return ( 
      <section className="cursos"> 
        <div> 
            <img src="/vite.svg" alt="Curso" /> 
            <h2>Nome do Curso</h2> 
        </div> 
      </section> 
  ) 
}  
export default Cursos

Observe nesse código que foi criada uma função com o mesmo nome do arquivo (cursos) e foi dado um return com o conteúdo HTML desejado. Depois, exporta-se a função com o código export default Cursos.

Nesse código, também é possível ver outra diferença importante do JSX em comparação ao HTML5. No JSX, a tag que não tem fechamento precisa obrigatoriamente ser encerrada internamente, inserindo o /> no final. É possível ver isso na tag img, que no HTML5 fecha normalmente sem a necessidade da barra (/), mas no JSX é preciso adicionar a barra.

O React, como já comentado anteriormente, possibilita a utilização de hooks, trazendo algumas facilidades, como resgatar os valores dos campos enviados via formulário HTML. Para exemplificar essa funcionalidade, veja a criação de um novo componente, que será chamado de Acesso.jsx.

Já no início do componente Acesso.jsx, será importado o hook useState do React para esse componente. A importação é feita desta forma: 

                      
import { useState } from "react"

Será necessário criar constantes com o nome do seu input e, junto a isso, uma constante que será atualizada conforme o valor do input. Por exemplo: 

                      
const [email, setEmail] = useState()

E vincula-se o evento change, que representa a alteração de valor de um campo (input) ao setEmail. Veja um exemplo: 

                        
<input onChange={(e) => setEmail(e.target.value)} type="email" name="email" />

Agora, toda vez que se atualiza ou insere um valor no campo email, é possível pegar esse valor com a constante email. Veja um exemplo completo com os campos email e senha, unindo ao evento de clique: 

                    
import { useState } from "react" 

function Acesso() { 
    const [email, setEmail] = useState() 
    const [senha, setSenha] = useState() 
    function enviarDados(e){ 
        e.preventDefault() 
        alert("Confira seus dados: " + email + " - " + senha) 
    } 
    return (                     
    <form>            
    <input onChange={(e) => setEmail(e.target.value) } type="email" name="email" placeholder="Seu E-mail" required />                 
    <input onChange={(e) => setSenha(e.target.value) } type="password" name="senha" placeholder="Sua Senha" required />                 
    <button onClick={enviarDados}>Acessar</button>                  
    </form>       
      )                      
  } 
export default Acesso

Agora basta importar esse componente ao App.js:

                    
import Acesso from './components/Acesso'

Inserindo CSS no Angular

Adicionar estilos CSS no Angular é bem simples. Basicamente, há dois modos de adicionar estilos: o estilo global, no qual o que for inserido vale para todo o projeto, e modo via componente, no qual o que é adicionado de estilo fica valendo exclusivamente para o componente, mesmo que tenham classes ou tags iguais.

Primeiramente, veja como adicionar um estilo global. Suponha que você quer mudar o tipo de fonte-padrão de todo o seu projeto Angular. Para isso, vá ao arquivo style.css, localizado na raiz da pasta “src”: 

                        
body{ 
    font-family: Verdana, Geneva, Tahoma, sans-serif; 
}

Isso mudará o tipo de fonte em todo o projeto. Mesmo que você crie mais componentes, esse estilo será aplicado a ele também.

Você deve ter percebido que, quando se cria um componente novo, em seus arquivos há também um arquivo de CSS, e se for colocado um estilo no arquivo CSS do componente, ele aplicará o estilo única e exclusivamente ao HTML desse componente. Veja um exemplo no arquivo boas-vindas.component.css: 

                        
h2{ 
    color: #5f9ea0; 
}   
h1{ 
    color: #8a2be2; 
    text-transform: uppercase; 
    font-size: 22px; 
}

Nesse exemplo, já havia sido criado um estilo para a tag h2. Agora, foi adicionado um estilo para a tag h1. No entanto, se você olhar o arquivo HTML do seu componente, verá que não foi adicionada nenhuma tag h1 ainda. Contudo, se verificar o arquivo principal do seu projeto (app.component.html), há uma tag h1, mas ela não sofrerá as modificações de estilo inseridas no CSS do componente.

Renderização condicional

Agora, você entenderá como criar conteúdos condicionais no Angular, ou seja, mostrar um conteúdo ou até mesmo realizar uma ação de acordo com alguma condição. Para exemplificar esse recurso, veja como criar uma mensagem na tela que será exibida apenas ao clicar (ou tocar) em um determinado conteúdo.

Comece criando um componente novo chamado alerta. Para isso, realize todo aquele processo de criação de componentes no Angular. Digite no terminal o ng generate component components/alerta e importe o novo componente no arquivo app.components.ts.

Para iniciar o processo de conteúdo condicional, vá ao arquivo alerta.component.ts e crie uma variável do tipo boolean, que servirá de parâmetro para mostrar ou não determinado conteúdo. 

                        
export class AlertaComponent { 
  mostrarOcultar: boolean = true 
}

Repare, nesse código, que foi criada a propriedade mostrarOcultar. Essa propriedade é do tipo boolean e seu valor inicial é igual a true.

Agora, no arquivo .html do componente alerta, insira seu conteúdo condicional: 

                        
@if(mostrarOcultar) { 
    <div>    
            <h2>Aviso Condicional</h2>    
            <p>Este conteúdo será mostrado de acordo com uma condição.</p> 
    </div>             
}

Observe que a sintaxe da versão mais atual do Angular para criar condicionais é muito similar à sintaxe do JavaScript tradicional. Aqui, só se acrescentou o arroba (@) antes do if.

<app-alerta></app-alerta>

O que ocorrerá agora é que a propriedade mostrarOcultar começa como false, mas, ao clicar no botão, será disparado o método mostrar() e mudará o valor da propriedade para true. Isso fará mostrar novamente a mensagem.

Porém, para que o exemplo fique mais completo e funcional, será usada uma sintaxe lógica do JavaScript que permite mudar o valor de um boolean sempre que for clicado. Para isso, será modificado o método mostrar para: 

                        
mostrar(): void { 
  this.mostrarOcultar = !this.mostrarOcultar 
}

Esse código indica que o valor da propriedade mostrarOcultar sempre será o oposto do valor atual, ou seja, se ele estiver como true, mudará para false e vice-versa. Isso faz com que o botão, ao ser clicado, primeiramente mostre a mensagem, mas, ao ser clicado novamente, oculte a mensagem, alternando assim entre os dois estados, visível e oculto.

Node.js

O Node.js, comumente chamado de Node, é uma plataforma de software de código aberto para a execução de código JavaScript no lado do servidor. Foi criado por Ryan Dahl em 2009 e é construído em cima do V8, que é o motor de JavaScript desenvolvido pelo Google para o navegador Chrome. O Node.js é projetado para construir aplicações de rede escaláveis e de alto desempenho, particularmente servidores web. Antes do Node.js, JavaScript era uma linguagem de programação voltada para client-side, executada apenas nos navegadores dos usuários. O Node.js expandiu o alcance do JavaScript, permitindo seu uso no lado do servidor, o que significa que os desenvolvedores podem escrever tanto o código do lado do cliente quanto do servidor em uma única linguagem

NPM (node package manager): o Node.js vem com o NPM, um gerenciador de pacotes que facilita a instalação, o compartilhamento e a atualização de pacotes de código. O NPM é uma parte essencial do ecossistema Node.js, fornecendo acesso a mais de 800 mil bibliotecas ou “pacotes” que podem ser facilmente integrados em projetos Node.js.

Apesar de ser popular para desenvolvimento web, o Node.js também é usado em uma variedade de outros contextos, incluindo automação de desktop, desenvolvimento de microsserviços, robótica e muito mais.

O Node.js transformou significativamente o desenvolvimento de software, permitindo uma abordagem mais unificada ao usar JavaScript em toda a stack (full stack) e continua sendo uma tecnologia fundamental para muitos desenvolvedores e empresas em todo o mundo.

Angular

AngularJS é um framework JavaScript open-source, desenvolvido inicialmente por Misko Hevery e Adam Abrons em 2009, e posteriormente mantido pelo Google. Foi criado para facilitar a construção e o teste de aplicações de página única (Single Page Applications - SPAs), onde, conforme vimos no capítulo sobre React, a maioria das atividades acontece em uma única página web, proporcionando uma experiência de usuário mais fluida e rápida.

Em 2016, o AngularJS evoluiu para uma nova plataforma chamada simplesmente de Angular (às vezes referido como Angular 2+), que é uma reescrita completa do AngularJS com melhor desempenho, uma arquitetura mais moderna e um melhor suporte para dispositivos móveis. Angular não é uma atualização direta do AngularJS, mas sim uma re-implementação que usa TypeScript e oferece funcionalidades melhoradas, como componentes web padrões e melhores práticas de desenvolvimento.

Assim como o React, para conseguir evoluir com o Angular é necessário ter domínio de HTML, CSS e JavaScript. E também será necessário ter o Node instalado, como editor também vamos utilizar o VS Code.

Dados dinâmicos e compartilhamento de dados

Com o Angular, é possível compartilhar dados do componente de nível superior (pai) com os componentes inferiores (filhos). Para exemplificar esse recurso, veja como criar um componente chamado cursos. Digite no terminal do VS Code o comando para criar componentes novos: 

              
ng generate component components/cursos

Esse comando gera, então, os arquivos para o novo componente dentro da pasta “components”. Agora, vá até o arquivo app.components.ts e importe o novo componente. Veja o arquivo completo e atualizado: 

              
  import { Component } from '@angular/core'; 
  import { RouterOutlet } from '@angular/router'; 
  import { BoasVindasComponent } from './components/boas-vindas/boas-vindas.component'; 
  import { CursosComponent } from './components/cursos/cursos.component'; 

@Component({ 
  selector: 'app-root', 
  standalone: true, 
  imports: [RouterOutlet, BoasVindasComponent, CursosComponent], 
  templateUrl: './app.component.html', 
  styleUrl: './app.component.css' 
}) 
export class AppComponent { 
  title = 'projeto-novo'; 
}

Para iniciar essa funcionalidade de dados dinâmicos e compartilhamento de dados, vá até o arquivo cursos.component.ts e importe um recurso do Angular chamado input. 

              
import { Component, Input } from '@angular/core';

Crie essas variáveis com a função input, a princípio com valor vazio, conforme o exemplo:

              
export class CursosComponent { 
  @Input() titulo = "" 
  @Input() imagemCurso = "" 
}

Agora você pode chamar essas propriedades dentro do HTML do seu componente cursos, no arquivo cursos.component.html: 

              
<img src="{{ imagemCurso }}" alt="{{ titulo }}"> 
<h3>{{ titulo }}</h3>

Para inserir esse componente em seu projeto com os dados dinâmicos, realiza-se a chamada dele dentro do app.component.html: 

                
<h1>Primeiro Projeto com Angular</h1> 
<app-boas-vindas></app-boas-vindas> 
<app-cursos titulo="CSS 3" imagem="teste"></app-cursos>

Para inserir imagens em seu projeto, crie uma pasta chamada “images” dentro do diretório “src/assets” e insira nela as imagens da sua interface. Então, atualize o arquivo HTML, chamando o componente três vezes:

<app-boas-vindas></app-boas-vindas> 
<div class="cursos"> 
    <app-cursos titulo="CSS 3" imagemCurso="assets/images/css.png"></app-cursos>

    <app-cursos titulo="PHP" imagemCurso="assets/images/php.png"></app-cursos>

Renderização condicional

Agora, você entenderá como criar conteúdos condicionais no Angular, ou seja, mostrar um conteúdo ou até mesmo realizar uma ação de acordo com alguma condição. Para exemplificar esse recurso, veja como criar uma mensagem na tela que será exibida apenas ao clicar (ou tocar) em um determinado conteúdo.

Comece criando um componente novo chamado alerta. Para isso, realize todo aquele processo de criação de componentes no Angular. Digite no terminal o ng generate component components/alerta e importe o novo componente no arquivo app.components.ts.

Para iniciar o processo de conteúdo condicional, vá ao arquivo alerta.component.ts e crie uma variável do tipo boolean, que servirá de parâmetro para mostrar ou não determinado conteúdo. 

              
export class AlertaComponent { 
  mostrarOcultar: boolean = true 
}

Repare, nesse código, que foi criada a propriedade mostrarOcultar. Essa propriedade é do tipo boolean e seu valor inicial é igual a true.

Agora, no arquivo .html do componente alerta, insira seu conteúdo condicional: 

              
@if(mostrarOcultar) { 
<div>    
  <h2>Aviso Condicional</h2>    
  <p>Este conteúdo será mostrado de acordo com uma condição.</p> 
</div>             
}

Observe que a sintaxe da versão mais atual do Angular para criar condicionais é muito similar à sintaxe do JavaScript tradicional. Aqui, só se acrescentou o arroba (@) antes do if.

Agora é preciso incluir o componente alerta dentro do HTML do app:

              
<app-alerta></app-alerta>

Ao visualizar a tela no navegador, ele deverá mostrar o conteúdo do componente alerta normalmente. Isso porque a propriedade mostrarOcultar está com o valor true. Ao mudar o valor para false, o conteúdo deve desaparecer.

Para adicionar uma condição de “senão”, no caso, o else, também é bem simples: basta adicionar o @else logo após o fechamento de if. Veja um exemplo, complementando o arquivo alerta.component.html:

} @else{ 
    <h2>Sem Avisos</h2> 
}

Nesse caso, se a variável mostrarOcultar estiver com valor false, então será mostrada a mensagem do else.

Agora veja como unir o conteúdo condicional a um evento do JavaScript. Para fazer isso, é preciso colocar o nome do evento entre parênteses e vinculá-lo a algum método. Será criada uma tag button no conteúdo e depois será criado o método no .ts do seu componente.

Para esse exemplo, será removido o else e acrescentada a tag que servirá de alvo para disparar uma função: 

              
@if(mostrarOcultar) { 
<div>                     
      <h2>Aviso Condicional</h2>                           
      <p>Este conteúdo será mostrado de acordo com uma condição.</p>             
</div> 
} 
<button (click)="mostrar()">Mostrar Aviso</button>

Agora vá ao arquivo alerta.component.ts e crie esse método chamado mostrar(): 

              
export class AlertaComponent { 
      mostrarOcultar: boolean = false 
      mostrar(): void { 
        this.mostrarOcultar = true 
      }                          
}

O que ocorrerá agora é que a propriedade mostrarOcultar começa como false, mas, ao clicar no botão, será disparado o método mostrar() e mudará o valor da propriedade para true. Isso fará mostrar novamente a mensagem.

Porém, para que o exemplo fique mais completo e funcional, será usada uma sintaxe lógica do JavaScript que permite mudar o valor de um boolean sempre que for clicado. Para isso, será modificado o método mostrar para: 

              
mostrar(): void { 
  this.mostrarOcultar = !this.mostrarOcultar 
}

Esse código indica que o valor da propriedade mostrarOcultar sempre será o oposto do valor atual, ou seja, se ele estiver como true, mudará para false e vice-versa. Isso faz com que o botão, ao ser clicado, primeiramente mostre a mensagem, mas, ao ser clicado novamente, oculte a mensagem, alternando assim entre os dois estados, visível e oculto.

Contudo, agora, complemente ainda mais sua interface, mudando dinamicamente a mensagem do botão. Para isso, primeiro, crie em sua função do componente a variável que conterá o texto do botão e, depois, crie uma condição para mudar o texto. Veja um exemplo: 

              
export class AlertaComponent { 
  mostrarOcultar: boolean = false 
  btAviso = "Mostrar Mensagem" 
  mostrar(): void { 
    this.mostrarOcultar = !this.mostrarOcultar 
    if(this.mostrarOcultar){ 
      this.btAviso = "Fechar Mensagem" 
    }else{ 
      this.btAviso = "Mostrar Mensagem" 
    } 
  } 
}

Agora é preciso inserir a propriedade btAviso no lugar do texto estático do botão, no .html do componente Aviso: 

              
@if(mostrarOcultar) { 
<div>              
      <h2>Aviso Condicional</h2>                     
      <p>Este conteúdo será mostrado de acordo com uma condição.</p> 
</div> 
} 
<button (click)="mostrar()">{{btAviso}}</button>

Agora você aprenderá alguns recursos do Angular aplicados a formulários. Um recurso essencial em qualquer linguagem ou framework é conseguir resgatar os valores preenchidos nos formulários HTML.

Então, para exemplificar esse recurso, veja como criar um outro componente, que será chamado de acessos. Para criar esse componente, basta repetir os processos de criação de componentes Angular mostrados anteriormente.

Após criar o componente, lembre-se de importá-lo no arquivo app.component.ts e já o chamar dentro de app.component.html. 

              
<app-acessos></app-acessos>

Para conseguir usar o recurso de obter os valores preenchidos nos campos dos formulários, primeiramente é preciso importar um recurso adicional do Angular em seu componente. Sendo assim, vá ao arquivo acessos.component.ts, importe o FormsModule e já crie uma propriedade chamada email: 

              
  import { Component } from '@angular/core'; 
  import { FormsModule } from '@angular/forms'; 
@Component({ 
  selector: 'app-acessos', 
  standalone: true, 
  imports: [FormsModule], 
  templateUrl: './acessos.component.html', 
  styleUrl: './acessos.component.css' 
}) 
export class AcessosComponent { 
  email: string = "" 
}

Agora, no arquivo .html do seu componente, crie um formulário e adicione um recurso do Angular chamado ngModel: 

                
<form> 
  <input type="email" name="email" placeholder="Seu E-mail" [(ngModel)]="email"> 
  <input type="submit" name="enviar"> 
</form> 
<div> 
  Seu email é: {{ email }} 
</div>

É importante destacar que o valor inserido em ngModel é exatamente o nome da propriedade criada na função anterior, neste caso, email.

Se tudo estiver funcionando corretamente, ao digitar qualquer valor no campo do seu formulário, ele deve escrever esse valor na div inserida.

Agora veja como criar mais um campo em seu formulário e como criar uma função para que os valores só apareçam na tela após clicar no botão de enviar do formulário.

No arquivo .ts do componente acessos, atualize sua class com novas propriedades e um método para mostrar os valores somente se os campos email e usuario estiverem preenchidos: 

              
export class AcessosComponent { 
  email: string = "" 
  usuario: string = "" 
  confirme: boolean = false 
  verDados(): void {     
    if(this.email && this.usuario){ 
      this.confirme = true 
    }else{ 
      this.confirme = false 
    } 
  } 
}
Já no acessos.component.html, inclua um novo campo e uma lógica condicional para mostrar os valores: 
              
<form> 
<input type="text" name="usuario" placeholder="Seu Nome" [(ngModel)]="usuario"> 
<input type="email" name="email" placeholder="Seu E-mail" [(ngModel)]="email"> 
<input type="submit" name="enviar" (click)="verDados()"> 
</form> 
@if (confirme){ 
<div> 
 <p>Bem vindo, {{ usuario }}</p> 
 <p>Você acessou com o e-mail: {{ email }}</p> 
</div> 
}

Perceba, em sua lógica, que o botão de enviar dispara a função verDados(), e, de acordo com o if, os dados só serão mostrados se a propriedade confirme estiver como true. O método verDados() só transforma a propriedade confirme em true se os valores dos campos email e usuario estiverem preenchidos.
