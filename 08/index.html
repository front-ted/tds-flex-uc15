Framework Bootstrap: recursos para mobile

O Bootstrap é um framework front-end gratuito e de código aberto utilizado para desenvolver sites e aplicações web responsivas e para dispositivos móveis. Foi originalmente desenvolvido pelo Twitter, tanto que, em suas primeiras versões, chamava-se Twitter Bootstrap. No entanto, atualmente o Bootstrap é um projeto totalmente à parte do Twitter (atualmente X).

O Bootstrap inclui uma coleção de ferramentas e componentes baseados em HTML HTML (hypertext markup language), CSS (cascading style sheets) e JS (JavaScript) que facilitam a criação de interfaces de usuário consistentes e modernas.

Esse framework vem há anos sendo uma das principais opções para o desenvolvimento de interfaces responsivas para a web, pois sua forma de integração é fácil e sua utilização é muito prática e eficaz. É amplamente utilizado não só por desenvolvedores independentes, mas por dezenas de projetos maiores.

Os principais CMS (content management system) e ferramentas web da atualidade utilizam o Bootstrap, por exemplo: WordPress, Joomla, OpenCart, Moodle e mais uma infinidade de softwares. A seguir, você aprenderá as principais funcionalidades desse excelente framework.

Instalando o Bootstrap

O processo de instalação do Bootstrap é bem simples: basta acessar o site oficial dele e baixar sua última versão estável.

Repare, ao acessar o site, que é possível instalar o Bootstrap via terminal, porém o processo para a instalação via download direto é bem mais simples. Então, ao clicar em Download, você será direcionado a outra página do site.

Haverá três opções para download: a versão completa e compacta com apenas os arquivos necessários para se trabalhar, uma versão com arquivos raiz para modificação e outra com exemplos.

Faça download da primeira opção, apenas com os arquivos necessários para começar a trabalhar com Bootstrap. O download virá compactado em arquivo único no formato ZIP. Clique com o botão direito do mouse nesse arquivo e extraia-o. Em seguida, copie as pastas CSS e JS para a pasta raiz do seu projeto.

Perceba que a estrutura do Bootstrap é bem simples. Há apenas duas pastas, e dentro delas há os arquivos necessários ao seu projeto. Olhando com mais atenção, ao abrir um dos diretórios, você perceberá que para cada arquivo há duas versões, a versão original e a versão “minificada”. Neste momento, você já deve ter percebido a importância de utilizar arquivos com o tamanho otimizado, então serão usadas sempre as versões que contêm o “min” no nome, pois, a princípio, nenhum arquivo do Bootstrap será modificado, e sim importado em seus projetos.

Então, o Bootstrap não trará nenhuma modificação em seus conteúdos HTML. Com ele, você criará seus arquivos HTML normalmente, estática ou dinamicamente, pois o Bootstrap apenas auxiliará na criação de estilos CSS e em alguns recursos de JavaScript. Inclusive, é possível integrar o Bootstrap a praticamente qualquer ferramenta, como React e Angular.

Para realizar a integração do Bootstrap, é preciso incluir seu arquivo CSS e JS principal em sua página. Veja um exemplo de HTML:

                        
                          <!DOCTYPE html>  
          
                          <html>  
          
                          <head> 
          
                          <meta charset="utf-8">  
          
                          <meta name="viewport" content="width=device-width, initial-scale=1">  
          
                          <title>Começando com Bootstrap</title> 
          
                          <link rel="stylesheet" type="text/css" href="css/bootstrap.min.css"> 
          
                          <script src="js/bootstrap.min.js" type="text/javascript"></script> 
          
                          </head> 
          
                          <body>  
          
                          </body> 
          
                          </html>

Perceba que foi incluído apenas o arquivo principal de cada pasta. É preciso ter atenção aos caminhos dos arquivos. Nesse caso, foram colocados apenas o diretório e o nome do arquivo, pois seu HTML está na raiz da pasta do projeto, com o nome index.html.

Agora que já foi integrado o arquivo CSS e JS do Bootstrap em seu HTML, comece a entender como funciona o conceito de container, pois ele é fundamental para estruturar e centralizar o conteúdo da página. Ele ajuda a criar layouts responsivos e consistentes ao definir margens automáticas e limites de largura.

Desenvolvendo layouts com Bootstrap

No Bootstrap, existem três variações de container. A primeira variação é o container tradicional, que aplica margens automáticas e centraliza o “bloco” de conteúdo. A segunda variação é o container-fluid, que ocupa 100% da largura disponível do viewport, independentemente do tamanho da tela. Ele é útil quando você precisa que o layout seja totalmente fluido e se ajuste ao tamanho do navegador sem limites de largura, como em slides ou banners.

Também é possível usar o container com os chamados breakpoints do Bootstrap, que é a terceira variação. Esses containers combinam as características do container tradicional com a responsividade do container-fluid. Veja alguns exemplos:

.container-sm: tem largura fixa a partir de 576 px.

.container-md: tem largura fixa a partir de 768 px.

.container-lg: tem largura fixa a partir de 992 px.

.container-xl: tem largura fixa a partir de 1.200 px.

.container-xxl: tem largura fixa a partir de 1.400 px.

Esses containers se comportam como container-fluid até atingirem o ponto de interrupção especificado, após o qual adquirem uma largura fixa.

No entanto, a forma mais utilizada na maioria das seções do layout da sua interface geralmente é o container tradicional, pois ele é fixo e tem larguras máximas predefinidas em diferentes pontos de interrupção (breakpoints) da responsividade. Ele centraliza automaticamente o conteúdo e aplica margens laterais, garantindo que o layout se ajuste bem em dispositivos de tamanhos variados. As larguras máximas são:

  • Até 576 px (extra small, XS): 100% de largura.
  • A partir de 576 px (small, SM): 540 px.
  • A partir de 768 px (medium, MD): 720 px.
  • A partir de 992 px (large, LG): 960 px.
  • A partir de 1.200 px (extra large, XL): 1.140 px.
  • A partir de 1.400 px (extra extra large, XXL): 1.320 px.

Você perceberá que é possível usar esses breakpoints em diversos outros recursos do Bootstrap, e as nomenclaturas serão sempre as mesmas.

Agora você entenderá o sistema de grids do Bootstrap. É por meio dele que se consegue posicionar os conteúdos em qualquer lugar do layout da sua interface. O sistema de grid do Bootstrap consiste em estilos predefinidos da propriedade flex do CSS. Porém, aqui não será preciso mexer em nenhum estilo CSS. Com esse sistema, é possível determinar as colunas do seu layout e seu comportamento apenas inserindo as classes predeterminadas do Bootstrap.

O sistema de grid do Bootstrap permite criar layouts responsivos de maneira fácil e eficiente. Ele é baseado em um sistema de 12 colunas, que pode ser ajustado para diferentes tamanhos de tela usando classes específicas.

Para que as grids funcionem corretamente, precisam estar dentro de um elemento superior com alguma classe de container. Então, cria-se a grid inserindo as classes para definir as linhas e as colunas da aplicação.

As linhas (row) são usadas para criar uma série de colunas e sempre devem estar dentro de um container. Já as colunas (col) são os elementos que contêm o conteúdo e são filhos diretos das linhas. As colunas são flexíveis e podem ser dimensionadas para ocupar diferentes proporções da linha.

O sistema de grid do Bootstrap usa classes predefinidas para criar colunas de tamanhos variados. Essas classes são baseadas nos breakpoints e podem ser combinadas para criar layouts complexos e responsivos.

Cada classe de coluna pode ser seguida por um número de 1 a 12 para especificar o número de colunas que o elemento deve ocupar. Por exemplo, .col-md-6 faz com que o elemento ocupe 6 das 12 colunas no breakpoint md.

O Bootstrap também permite, via classes, mudar o alinhamento das colunas na horizontal. Neste caso, foi colocada a classe na mesma tag da classe row. Para essa funcionalidade, o framework tem seis classes diferentes.

justify-content-start: é o valor padrão, que indica que os elementos estarão alinhados da esquerda para a direita.

justify-content-end: inverte o alinhamento, indicando que os elementos ficarão alinhados da direita para a esquerda.

justify-content-center: centraliza os elementos.

justify-content-between: distribui os elementos com espaçamentos iguais entre eles. No entanto, o primeiro e último elemento de cada linha ficam colados em suas extremidades, ou seja, não há margens horizontais entre o início e fim de cada linha.

justify-content-around: distribui os elementos com o mesmo espaçamento entre eles, incluindo as extremidades da linha.

justify-content-evenly: distribui os elementos de forma que o espaçamento entre eles, incluindo o espaço entre as bordas do elemento pai, seja igual.

Perceba que essas classes, basicamente, adotam os valores da propriedade justify-content de elementos flex presentes no CSS3.

Com o Bootstrap, também é possível mudar a ordem das colunas de um modo muito prático, apenas inserindo a classe order- seguida da posição a ser ocupada. Essa funcionalidade segue exatamente o mesmo preceito da propriedade order dos layouts Flexbox do CSS 3.

É comum se confundir nessa funcionalidade, pois, intuitivamente, se definido o estilo order-2, entende-se que ele deveria ir para a segunda posição. Mas isso não ocorre porque, por padrão, todo o elemento começa com order zero. Um dos recursos mais completos e complexos do HTML5, é a criação de formulários. Formatar a visualização de qualquer formulário é algo necessário, porém bastante trabalhoso.

Um dos principais campos de entrada de qualquer formulário HTML são os campos com a tag input, porém se você já tem conhecimentos sólidos em HTML5, já sabe que há vários tipos de campos que usam a tag input. Instalando o Bootstrap

O processo de instalação do Bootstrap é bem simples: basta acessar o site oficial dele e baixar sua última versão estável.

Repare, ao acessar o site, que é possível instalar o Bootstrap via terminal, porém o processo para a instalação via download direto é bem mais simples. Então, ao clicar em Download, você será direcionado a outra página do site.

Haverá três opções para download: a versão completa e compacta com apenas os arquivos necessários para se trabalhar, uma versão com arquivos raiz para modificação e outra com exemplos.

Faça download da primeira opção, apenas com os arquivos necessários para começar a trabalhar com Bootstrap. O download virá compactado em arquivo único no formato ZIP. Clique com o botão direito do mouse nesse arquivo e extraia-o. Em seguida, copie as pastas CSS e JS para a pasta raiz do seu projeto.

Perceba que a estrutura do Bootstrap é bem simples. Há apenas duas pastas, e dentro delas há os arquivos necessários ao seu projeto. Olhando com mais atenção, ao abrir um dos diretórios, você perceberá que para cada arquivo há duas versões, a versão original e a versão “minificada”. Neste momento, você já deve ter percebido a importância de utilizar arquivos com o tamanho otimizado, então serão usadas sempre as versões que contêm o “min” no nome, pois, a princípio, nenhum arquivo do Bootstrap será modificado, e sim importado em seus projetos.

Então, o Bootstrap não trará nenhuma modificação em seus conteúdos HTML. Com ele, você criará seus arquivos HTML normalmente, estática ou dinamicamente, pois o Bootstrap apenas auxiliará na criação de estilos CSS e em alguns recursos de JavaScript. Inclusive, é possível integrar o Bootstrap a praticamente qualquer ferramenta, como React e Angular.

Para realizar a integração do Bootstrap, é preciso incluir seu arquivo CSS e JS principal em sua página. Veja um exemplo de HTML:

                
                  <!DOCTYPE html>  
  
                  <html>  
  
                  <head> 
  
                  <meta charset="utf-8">  
  
                  <meta name="viewport" content="width=device-width, initial-scale=1">  
  
                  <title>Começando com Bootstrap</title> 
  
                  <link rel="stylesheet" type="text/css" href="css/bootstrap.min.css"> 
  
                  <script src="js/bootstrap.min.js" type="text/javascript"></script> 
  
                  </head> 
  
                  <body>  
  
                  </body> 
  
                  </html>

Perceba que foi incluído apenas o arquivo principal de cada pasta. É preciso ter atenção aos caminhos dos arquivos. Nesse caso, foram colocados apenas o diretório e o nome do arquivo, pois seu HTML está na raiz da pasta do projeto, com o nome index.html.

Agora que já foi integrado o arquivo CSS e JS do Bootstrap em seu HTML, comece a entender como funciona o conceito de container, pois ele é fundamental para estruturar e centralizar o conteúdo da página. Ele ajuda a criar layouts responsivos e consistentes ao definir margens automáticas e limites de largura.

Desenvolvendo layouts com Bootstrap

No Bootstrap, existem três variações de container. A primeira variação é o container tradicional, que aplica margens automáticas e centraliza o “bloco” de conteúdo. A segunda variação é o container-fluid, que ocupa 100% da largura disponível do viewport, independentemente do tamanho da tela. Ele é útil quando você precisa que o layout seja totalmente fluido e se ajuste ao tamanho do navegador sem limites de largura, como em slides ou banners.

Também é possível usar o container com os chamados breakpoints do Bootstrap, que é a terceira variação. Esses containers combinam as características do container tradicional com a responsividade do container-fluid. Veja alguns exemplos:

.container-sm: tem largura fixa a partir de 576 px.

.container-md: tem largura fixa a partir de 768 px.

.container-lg: tem largura fixa a partir de 992 px.

.container-xl: tem largura fixa a partir de 1.200 px.

.container-xxl: tem largura fixa a partir de 1.400 px.

Esses containers se comportam como container-fluid até atingirem o ponto de interrupção especificado, após o qual adquirem uma largura fixa.

No entanto, a forma mais utilizada na maioria das seções do layout da sua interface geralmente é o container tradicional, pois ele é fixo e tem larguras máximas predefinidas em diferentes pontos de interrupção (breakpoints) da responsividade. Ele centraliza automaticamente o conteúdo e aplica margens laterais, garantindo que o layout se ajuste bem em dispositivos de tamanhos variados. As larguras máximas são:

  • Até 576 px (extra small, XS): 100% de largura.
  • A partir de 576 px (small, SM): 540 px.
  • A partir de 768 px (medium, MD): 720 px.
  • A partir de 992 px (large, LG): 960 px.
  • A partir de 1.200 px (extra large, XL): 1.140 px.
  • A partir de 1.400 px (extra extra large, XXL): 1.320 px.

Você perceberá que é possível usar esses breakpoints em diversos outros recursos do Bootstrap, e as nomenclaturas serão sempre as mesmas.

Agora você entenderá o sistema de grids do Bootstrap. É por meio dele que se consegue posicionar os conteúdos em qualquer lugar do layout da sua interface. O sistema de grid do Bootstrap consiste em estilos predefinidos da propriedade flex do CSS. Porém, aqui não será preciso mexer em nenhum estilo CSS. Com esse sistema, é possível determinar as colunas do seu layout e seu comportamento apenas inserindo as classes predeterminadas do Bootstrap.

O sistema de grid do Bootstrap permite criar layouts responsivos de maneira fácil e eficiente. Ele é baseado em um sistema de 12 colunas, que pode ser ajustado para diferentes tamanhos de tela usando classes específicas.

Para que as grids funcionem corretamente, precisam estar dentro de um elemento superior com alguma classe de container. Então, cria-se a grid inserindo as classes para definir as linhas e as colunas da aplicação.

As linhas (row) são usadas para criar uma série de colunas e sempre devem estar dentro de um container. Já as colunas (col) são os elementos que contêm o conteúdo e são filhos diretos das linhas. As colunas são flexíveis e podem ser dimensionadas para ocupar diferentes proporções da linha.

O sistema de grid do Bootstrap usa classes predefinidas para criar colunas de tamanhos variados. Essas classes são baseadas nos breakpoints e podem ser combinadas para criar layouts complexos e responsivos.

Cada classe de coluna pode ser seguida por um número de 1 a 12 para especificar o número de colunas que o elemento deve ocupar. Por exemplo, .col-md-6 faz com que o elemento ocupe 6 das 12 colunas no breakpoint md.

Criando formulários com Bootstrap

Um dos recursos mais completos e complexos do HTML5, é a criação de formulários. Formatar a visualização de qualquer formulário é algo necessário, porém bastante trabalhoso.

Contudo, o Bootstrap traz uma série de recursos para formar estilos de formulários modernos e responsivos, de uma maneira muito prática. Para isso, veja como construir formulários usando o esquema de layout do Bootstrap e utilizar algumas classes predefinidas específicas para os campos dos formulários.

Um dos principais campos de entrada de qualquer formulário HTML são os campos com a tag input, porém se você já tem conhecimentos sólidos em HTML5, já sabe que há vários tipos de campos que usam a tag input. Basicamente, ao mudar o atributo type de cada campo, o navegador traz um campo com funcionalidades e formatos bem distintos.

Então é comum qualquer formulário ter diversos campos com a tag input. E para uma formatação básca, porém responsiva e moderna, com Bootstrap, basta adicionar a classe form-control em seus campos.

Veja um exemplo:

              
                <form> 

                  <input class="form-control" type="text" name="nome" placeholder="Seu Nome" > 
                  
                  </form>

Conhecendo componentes Bootstrap

Os componentes do Bootstrap são elementos de interface pré-estilizados que você pode usar para construir rapidamente páginas e aplicações web com um design consistente e moderno. Esses componentes são baseados em HTML, CSS e JavaScript e são projetados para serem responsivos e acessíveis por padrão.

Offcanvas e Navbar

O componente Offcanvas do Bootstrap é um recurso essencial para qualquer site responsivo. Ele permite que você tenha conteúdos HTML escondidos em sua interface, mas que podem ser visualizados clicando em um elemento.

Esse componente traz o conteúdo na tela de forma estática sobre os demais elementos, normalmente abrindo, com um efeito de transição, na lateral da sua interface, mas ocupando a altura inteira da tela do dispositivo.

É muito comum criar esse tipo de recurso em menus mobile, mas o Offcanvas permite inserir qualquer conteúdo HTML nele. Veja um exemplo:

                
                  <a class="btn btn-primary" data-bs-toggle="offcanvas" href="#offcanvasExemplo" role="button" aria-controls="offcanvasExemplo"> 

                    Abrir Conteúdo 
                
                  </a>   
                
                  <div class="offcanvas offcanvas-start" tabindex="-1" id="offcanvasExemplo" aria-labelledby="offcanvasExemploLabel"> 
                
                    <div class="offcanvas-header"> 
                
                      <h5 class="offcanvas-title" id="offcanvasExemploLabel">Offcanvas</h5> 
                
                      <button type="button" class="btn-close" data-bs-dismiss="offcanvas" aria-label="Close"></button> 
                
                    </div> 
                
                    <div class="offcanvas-body"> 
                
                      <div> 
                
                        Conteúdo do seu componente 
                
                      </div>     
                
                    </div> 
                
                  </div>

Repare que é preciso ter um elemento acionador, como um botão ou link, e o desenvolvimento do conteúdo que ficará no Offcanvas em si.

Por padrão, o conteúdo aparece na lateral esquerda da sua interface, mas é possível mudar sua posição por meio de algumas classes predeterminadas.

  • offcanvas-start: coloca o Offcanvas à esquerda da janela de visualização.
  • offcanvas-end: coloca o Offcanvas à direita da janela de visualização.
  • offcanvas-top: coloca o Offcanvas no topo da janela de visualização.
  • offcanvas-bottom: coloca o Offcanvas na parte inferior da janela de visualização.

Essas classes de posicionamento devem ser adicionadas na div principal da estrutura do conteúdo do componente.

                
                  <div class="offcanvas offcanvas-top" id="offcanvasExemplo" aria-labelledby="offcanvasExemploLabel">
