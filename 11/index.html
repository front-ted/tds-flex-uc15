Testes em aplicações web: teste de script, compatibilidade com navegadores, adequações para responsividade, ferramentas automatizadas de teste front-end

Realizar testes em aplicações web é crucial para garantir que o site funcione corretamente em diferentes ambientes e para todos os usuários. Alguns dos principais testes envolvem realizar testes de scripts, verificar a compatibilidade da sua interface entre os diversos navegadores e verificar e realizar adequações de responsividade em diferentes dispositivos. No entanto, também existem ferramentas que automatizam alguns desses testes no front-end da sua interface.

Teste de script

Quando se fala em testes de script, eles geralmente envolvem a verificação de scripts JavaScript em uma aplicação web para garantir que eles funcionem corretamente. Isso pode incluir testes de unitários, de integração e end-to-end (E2E).

Os testes de script são fundamentais para garantir a qualidade do software e a funcionalidade correta das aplicações web.

Testes unitários

Verificam a menor parte testável do software, conhecida como “unidade”. Em geral, isso significa testar funções ou métodos individuais em isolamento para garantir que funcionem conforme o esperado.

O objetivo desse teste é assegurar que cada parte do código funcione corretamente de forma independente.

Os testes unitários são executados isoladamente, sem dependências externas como bancos de dados ou serviços externos e são geralmente rápidos de executar. Também são usados para alcançar alta cobertura de código, detectando erros em partes específicas do código.

Uma das principais ferramentas para testes unitários em qualquer ambiente se chama Jasmine.

Jasmine

O Jasmine é um framework de testes unitários popular para JavaScript. Ele é baseado na metodologia de desenvolvimento guiado por comportamento (BDD – behavior driven development). Ele permite que você escreva testes claros e concisos que descrevem o comportamento esperado do seu código. O Jasmine foi desenvolvido em JavaScript puro, o que o torna independente de qualquer outro framework. Isso significa que você pode usá-lo com qualquer biblioteca ou projeto JavaScript.

Para começar a usar o Jasmine, você precisará instalá-lo em seu projeto. Apesar de conseguir executar seus testes direto no navegador, o site da ferramenta indica que é aconselhável rodar em um servidor JavaScript, pois alguns navegadores limitam as informações de erros fornecidas a scripts executados com base em URLs (uniform resource locator).

Então, tendo um servidor JavaScript configurado, você pode instalar o Jasmine com o seguinte comando NPM (node package manager):

                          
                            npm install jasmine

Depois de instalar o Jasmine, você pode começar a escrever seus testes. Um teste unitário típico do Jasmine consiste em três partes:

  • Descrição: a descrição descreve o que o teste está testando.
  • Configuração: a configuração define o cenário para o teste.
  • Expectativas: as expectativas definem o que você espera que o código faça.

Veja um exemplo de teste direto no código:

                      
                        describe("Uma função que soma dois números", function() { 

                          it("deve retornar a soma de dois números", function() { 
                      
                            expect(soma(1, 2)).toBe(3); 
                      
                          }); 
                      
                        });

Esse teste testa uma função chamada soma, que deve retornar a soma de dois números. O teste configura a função para ser chamada com os números 1 e 2 e, em seguida, verifica se a função retorna o resultado esperado de 3.

O Jasmine é uma ferramenta poderosa e fácil de usar que pode ajudá-lo a escrever testes unitários de alta qualidade para seu código JavaScript. Ao usar o Jasmine, você pode melhorar a qualidade do seu código, aumentar sua confiança no código e facilitar a refatoração.

Realizar testes unitários é crucial para o desenvolvimento web de qualidade, pois oferece diversos benefícios, como identificar e corrigir bugs no código precocemente e facilitar a refatoração, pois os testes unitários servem de rede de segurança que permite refatorar o código sem medo de introduzir novos bugs. Isso torna o código mais flexível e fácil de manter no longo prazo.

Testes de integração

Enquanto os testes unitários focam unidades isoladas, os testes de integração verificam o funcionamento conjunto de diferentes módulos ou componentes do sistema. O objetivo é garantir que as unidades se integrem corretamente e que a comunicação entre elas esteja funcionando como esperado. Essa camada de testes garante que as interfaces entre os componentes estejam bem definidas e que o sistema funcione como um todo coeso.

Com os testes de integração, é possível detectar falhas de integração, identificando problemas na comunicação entre os componentes e evitando falhas inesperadas no sistema completo.

Esses testes também garantem que as interfaces entre os componentes estejam bem definidas e consistentes, promovendo, assim, a integração harmoniosa dos componentes, evitando comportamentos inesperados. Isso faz aumentar a confiança no funcionamento do sistema como um todo.

Testes E2E

Os testes E2E (end-to-end) simulam o fluxo completo de uso de uma aplicação, desde a interação inicial do usuário até o resultado final. O objetivo é garantir que o sistema funcione conforme o esperado do ponto de vista do usuário, validando sua experiência completa.

Os testes E2E testam o fluxo completo da aplicação, do início ao fim, e têm o foco na experiência do usuário, pois validam se a aplicação atende às expectativas dele.

Esses testes são mais complexos, então exigem mais tempo e esforço para serem escritos e executados. Por isso, podem ser lentos de executar, especialmente para aplicações complexas.

No entanto, seu uso traz muitos benefícios, como:

  • Detecção de falhas de usabilidade: identificam problemas que afetam a experiência do usuário, como interfaces confusas ou bugs que impedem a conclusão de tarefas.
  • Validação do fluxo de trabalho: garantem que o fluxo de trabalho da aplicação esteja funcionando como esperado do início ao fim.
  • Maior confiança do usuário: aumentam a confiança do usuário na qualidade e confiabilidade da aplicação.
  • Melhor experiência do usuário: contribuem para uma experiência do usuário mais positiva e satisfatória.

Atualmente, existem muitas ferramentas que auxiliam os testes E2E, e uma das mais populares se chama Selenium, cuja base é a automação de navegação web, permitindo testar aplicações web de forma abrangente e em diversos navegadores. O Selenium oferece suporte a diversas linguagens de programação, como Java, Python, C# e JavaScript, proporcionando flexibilidade para os desenvolvedores.

Porém, também há ferramentas populares mais específicas, como o Cypress, que se destaca como uma ferramenta de teste E2E moderna e com foco na experiência do desenvolvedor. Sua arquitetura inovadora baseada em JavaScript oferece uma experiência de desenvolvimento rápida e intuitiva. O Cypress é ideal para equipes que desejam testes E2E fáceis de escrever e depurar.

Compatibilidade com navegadores

No mundo dinâmico da web, onde diversos navegadores e dispositivos coexistem, garantir que seu site ou aplicativo funcione corretamente em todos eles é crucial para oferecer uma experiência consistente e positiva a seus usuários. É nesse contexto que entram os testes de compatibilidade com navegadores, também conhecidos como cross-browser testing.

Basicamente esses testes consistem em verificar como o seu site ou aplicativo se comporta em diferentes navegadores e dispositivos, identificando possíveis problemas de renderização, funcionalidade e usabilidade. O objetivo é garantir que, independentemente do navegador ou dispositivo utilizado, a experiência do usuário seja a mesma e atenda às suas expectativas.

Garantir que seu site ou sistema funcione em todos os dispositivos e navegadores atuais traz muitos benefícios a sua aplicação, como:

Melhorar a experiência do usuário

Um site ou aplicativo que funciona corretamente em todos os navegadores oferece uma experiência mais agradável e satisfatória para os usuários, aumentando a fidelidade e o engajamento.

Evitar problemas de usabilidade

Os testes de compatibilidade podem identificar problemas como interfaces desorganizadas, menus inacessíveis ou recursos que não funcionam em determinados navegadores, evitando que os usuários fiquem frustrados e abandonem o site.

Diminuir a taxa de rejeição

Um site compatível com diversos navegadores reduz a probabilidade de que os usuários se deparem com erros ou problemas de visualização, diminuindo a taxa de rejeição e aumentando o tempo que eles passam no site, impactando diretamente o SEO (search engine optimization).

O processo mais simples que se deve fazer é realizar testes manuais, o que envolve abrir o site ou sistema em diferentes navegadores e dispositivos para verificar se tudo funciona corretamente. Essa abordagem pode ser demorada e trabalhosa, mas permite uma análise mais profunda da experiência do usuário.

No entanto, existem ferramentas on-line, como BrowserStack, Sauce Labs, CrossBrowserTesting e Browserling, que auxiliam nesse processo, permitindo que você visualize seu site em diferentes navegadores sem precisar instalá-los.

No momento que você está desenvolvendo sua interface, é importante verificar se o estilo CSS, a tag HTML ou a função JavaScript que você está utilizando tem compatibilidade com os navegadores e dispositivos que o seu usuário final utilizará.

Uma ferramenta que auxilia muito nisso é o Can I Use. Ele mostra para você se o estilo ou a tag utilizada é compatível com determinados navegadores e dispositivos.

Adequações para responsividade

Atualmente existem diversas práticas e técnicas usadas para garantir que uma interface para desenvolvimento web seja eficaz em diferentes dispositivos e tamanhos de tela. Todas elas são abordadas no conhecimento sobre CSS. Relembre algumas técnicas:

Clique ou toque para visualizar o conteúdo.

Layout flexível ou fluid

Utilizar unidades de medida flexíveis como porcentagens em vez de unidades fixas (pixels) para larguras de contêineres e elementos, permitindo que eles se ajustem dinamicamente ao tamanho da tela.

Media queries

Utilizar media queries em CSS para aplicar estilos diferentes com base nas características da tela, como largura, altura e orientação. Isso permite ajustar o layout e o design conforme necessário para diferentes dispositivos.

Grids e frameworks responsivos

Utilizar sistemas de grid responsivos, como o Bootstrap, que facilitam a criação de layouts adaptáveis, ajustando automaticamente a disposição dos elementos com base no tamanho da tela.

Fontes e tipografia responsivas

Utilizar unidades de medida relativas como “em”, “rem” e “%” para o tamanho da fonte, garantindo que o texto seja legível em diferentes dispositivos sem precisar de zoom.

Desempenho e velocidade

Considerar o desempenho e a velocidade de carregamento, otimizando imagens e recursos para diferentes dispositivos, garantindo uma experiência de usuário rápida e eficiente.

Acessibilidade

Certificar-se de que o design responsivo não comprometa a acessibilidade, garantindo que usuários com necessidades especiais possam navegar e interagir com o site sem dificuldades adicionais.

Testes em dispositivos múltiplos

Testar regularmente o design em uma variedade de dispositivos e tamanhos de tela para garantir que a experiência do usuário seja consistente e eficaz em todas as plataformas.

Algumas das ferramentas mais populares e eficazes atualmente são o Selenium e Cypress.

Clique ou toque para visualizar o conteúdo.

Selenium

O Selenium é uma ferramenta de automação de testes muito popular e poderosa utilizada para testar aplicações web. O Selenium permite aos desenvolvedores e testadores escreverem scripts em várias linguagens de programação (como JavaScript, Python, C# etc.) para automatizar a interação com o navegador web. Com ele, é possível simular interações do usuário, como cliques, preenchimento de formulários, navegação entre páginas e validação de resultados, tudo de forma automatizada. Essa capacidade faz do Selenium uma ferramenta essencial para garantir a qualidade e o desempenho de aplicações web por meio de testes automatizados.

Começar a utilizar o Selenium é muito simples. Veja um exemplo com base na linguagem JavaScript. Então, no site oficial, clique em Downloads e clique em JavaScript. Lá haverá as instruções para a instalação. O processo é bem prático, mas você precisa ter o Node.js instalado em seu computador.

O Node é uma plataforma de software de código aberto para a execução de código JavaScript no lado do servidor, projetado para construir aplicações de rede escaláveis e de alto desempenho, particularmente servidores web. O Node expandiu o alcance do JavaScript, permitindo seu uso no lado do servidor, o que significa que os desenvolvedores podem escrever tanto o código do lado do cliente quanto do servidor em uma única linguagem.

Com o Node previamente instalado, crie um diretório para o seu projeto e abra-o no VS Code. Então, digite no terminal o seguinte comando para instalar o Selenium:

npm install selenium-webdriver

Então, ele criará todas as dependências e os arquivos necessários ao funcionamento do Selenium.

Instalação do Selenium
Fonte: Senac EAD (2024)

A figura mostra o terminal do VS Code, com o código de instalação do Selenium executado.

No exemplo dessa figura, foi criado o diretório Selenium e realizada a instalação nele.

Agora, no mesmo diretório, será criado um arquivo de JavaScript para executar as tarefas automatizadas. Neste exemplo, foi criado o arquivo teste.js. Já no início do arquivo, é preciso importar os módulos necessários, no caso, o webdriver do Selenium e o webdriver do navegador em que você deseja executar o teste.

Veja um exemplo levando em consideração o navegador Chrome:

                          
                            const { Builder, By, Key, until } = require('selenium-webdriver'); 
        
                            const chrome = require('selenium-webdriver/chrome');

No mesmo arquivo, logo após a importação dos módulos, cria-se uma função para executar as ações:

                          
                            async function exampleTest() { 
        
                              // Inicializar o WebDriver 
                          
                              let driver = await new Builder() 
                          
                                  .forBrowser('chrome') 
                          
                                  .build(); 
                          
                              try { 
                          
                                  // Abrir uma página 
                          
                                  await driver.get('https://www.google.com'); 
                          
                                  // Encontrar o campo de busca pelo nome q 
                          
                                  let element = await driver.findElement(By.name('q')); 
                          
                                  // Digitar texto no elemento 
                          
                                  await element.sendKeys('Hello, Selenium!', Key.RETURN); 
                          
                                  // Esperar até que a página carregue completamente 
                          
                                  await driver.wait(until.titleIs('Example Domain'), 1000); 
                          
                           
                          
                                  // Verificar o título da página 
                          
                                  let title = await driver.getTitle(); 
                          
                                  console.log('Page title is:', title); 
                          
                              } finally { 
                          
                                  // Fechar o navegador no final do teste 
                          
                                  await driver.quit(); 
                          
                              } 
                          
                          }

Nesse código, primeiro foi indicado que as ações seriam executadas no navegador Chrome. Depois, pede-se a ele que abra o site do Google. Ao acessar esse site, ele procura um campo com nome “q”, ou seja, uma tag input com o atributo name igual ao valor q (no site do Google, o campo da busca se chama “q”). Com o método sendkeys, pede-se a ele que digite, no campo encontrado, a frase “Hello, Selenium”.

Após o teste concluído, ele fecha a janela do navegador automaticamente com a função driver.quit().

Porém, para executar esse comando, é preciso usar um comando do Node. Basta escrever a palavra “node” seguida do caminho do arquivo a ser executado:

                          
                            node teste.js

Assim que o comando rodar, ele executará todas as ações descritas no código, abrir o navegador, acessar o site do Google, escrever o termo “Hello, Selenium” no campo de busca e, por último, fechar o navegador.

Perceba que, com o método findElement, é possível encontrar elementos em sua interface, atribuindo-os a variáveis, e executar ações neles. É claro que, neste caso, você precisa entender a estrutura do site a ser testado.

Por exemplo, suponha que você tem um site com cadastro, no qual é preciso preencher um campo de e-mail e senha e, então, clicar no botão de enviar. O primeiro passo é identificar esses elementos. Veja: 

                          
                            let nome = await driver.findElement(By.name('mauticform[nome]')); 
        
                            let email = await driver.findElement(By.name('mauticform[email]')); 
        
                            let enviar = await driver.findElement(By.className('mauticform-button'));

Observe que são obtidos os campos via name; já o botão busca-se via classe. O próximo passo é declarar as ações a serem executadas nesses campos.

                          
                            await nome.sendKeys('Teste de Oliveira', Key.RETURN); 
        
                            await email.sendKeys('teste@email.com', Key.RETURN); 
        
                            await enviar.click();

Nesse exemplo, escreva o termo “Teste de Oliveira” no campo nome. Também será escrito o e-mail “teste@email.com” no campo email. Por fim, será executado o evento de clique no elemento enviar.

Para testar essas ações, execute em um site de exemplo que contenha os elementos mencionados. Veja o código completo:

                          
                            const { Builder, By, Key, until } = require('selenium-webdriver'); 
        
                            const chrome = require('selenium-webdriver/chrome'); 
        
                            async function exampleTest() {     
        
                           let driver = await new Builder() 
        
                .forBrowser('chrome')       
        
                .build(); 
        
            try { 
        
                // Abrir uma página 
        
                await driver.get('https://profroberto.com.br/cadastro-aula'); 
        
        //Aguarda 2 segundos para a página carregar 
        
               await driver.manage().setTimeouts({implicit: 2000}); 
        
                // Encontrar os elementos na página 
        
                let nome = await driver.findElement(By.name('mauticform[nome]')); 
        
                let email = await driver.findElement(By.name('mauticform[email]')); 
        
                let enviar = await driver.findElement(By.className('mauticform-button')); 
        
                // Executar eventos 
        
                await nome.sendKeys('teste de Oliveira', Key.RETURN); 
        
                await email.sendKeys('teste@email.com', Key.RETURN); 
        
                await enviar.click();         
        
            } finally { 
        
                // Fechar o navegador no final do teste 
        
                await driver.quit(); 
        
            } 
        
        } 
        
              // Chamar a função 
        
              exampleTest();

Cypress

O Cypress tem uma API (application programming interface) simples e fluente, projetada especificamente para testar aplicações web modernas. Ele utiliza seletores diretamente no DOM (modelo de objeto de documento) e oferece comandos encadeados que são fáceis de ler e escrever. Ele executa testes de forma mais rápida e previsível devido ao seu controle mais direto sobre o ambiente. Além disso, pode simular interações do usuário de maneira mais eficiente, pois opera diretamente no mesmo contexto que a aplicação.

Começar a trabalhar com o Cypress não é muito diferente do Selenium. Você precisa basicamente das mesmas ferramentas, o Node.js e um editor como o VS Code.

Crie, então, um diretório para o seu ambiente de testes e abra-o no VS Code. Em seguida, é preciso instalar o Cypress via terminal:

                          
                            npm install cypress --save-dev

No Cypress, os testes são escritos usando uma sintaxe baseada em BDD (behavior driven development). Ele executa os testes diretamente no navegador que controla, o que facilita a depuração e a interação com a aplicação.

Crie um teste idêntico ao do tópico anterior. Abra o Google, faça uma pesquisa e verifique o título da página:

                          
                            describe('Teste de Busca no Google', () => { 
        
                              it('Visitar o site do Google e Buscar algo', () => { 
                            
                                // Visitar o site do Google 
                            
                                cy.visit('https://www.google.com'); 
                            
                             
                            
                                // Encontrar o campo de busca e digitar a consulta 
                            
                                cy.get('textarea[name="q"]').type('Cypress.io{enter}'); 
                            
                             
                            
                                // Verificar o título da página após a busca 
                            
                                cy.title().should('include', 'Cypress.io'); 
                            
                              }); 
                            
                            });

Perceba que a sintaxe é bem intuitiva, cy.visit indica a URL a ser acessada, cy.get busca o elemento e type efetua a busca.

                        cy.title().should('include', 'Cypress.io'); 
                    
                      }); 
                    
                    });

Perceba que a sintaxe é bem intuitiva, cy.visit indica a URL a ser acessada, cy.get busca o elemento e type efetua a busca.

Encerramento

As ferramentas automatizadas de teste front-end oferecem uma ampla gama de funcionalidades para garantir que suas aplicações web sejam robustas, funcionais e livres de bugs. Utilizar uma combinação dessas ferramentas pode proporcionar uma cobertura de teste mais abrangente e eficiente.
